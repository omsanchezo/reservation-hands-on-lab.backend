# Domain Entities Reference

Complete reference for generating APSYS domain entity classes.

---

## Base Class: AbstractDomainObject

All entities **must** inherit from `AbstractDomainObject`.

This base class provides:

| Member         | Type       | Behavior                                      |
|----------------|------------|-----------------------------------------------|
| `Id`           | `Guid`     | Auto-generated on creation. Never set manually. |
| `CreationDate` | `DateTime` | Auto-assigned (UTC). Never set manually.        |
| `IsValid()`    | `bool`     | Runs the validator and returns validity.        |
| `Validate()`   | `void`     | Runs the validator; throws on failure.          |
| `GetValidator()`| `IValidator` | Abstract. Must be overridden in every entity. |

---

## Namespace Convention

```
{project}.domain.entities
```

Example: `apsys.erp.domain.entities`

---

## Property Rules

### All properties must be `virtual`

NHibernate requires `virtual` for lazy loading and change tracking. There are **no exceptions**.

```csharp
public virtual string Name { get; set; } = string.Empty;
```

### Default initialization by type

| Type              | Default Value       |
|-------------------|---------------------|
| `string`          | `string.Empty`      |
| `int`             | (implicit `0`)      |
| `bool`            | (implicit `false`)  |
| `DateTime`        | (implicit default)  |
| `IList<T>`        | `new List<T>()`     |
| Entity reference  | `= null!`           |

### Example property declarations

```csharp
public virtual string Email { get; set; } = string.Empty;
public virtual string Name { get; set; } = string.Empty;
public virtual bool Locked { get; set; }
public virtual int Quantity { get; set; }
public virtual DateTime IssueDate { get; set; }
public virtual IList<Role> Roles { get; set; } = new List<Role>();
public virtual Customer Customer { get; set; } = null!;
```

## Constructors: Exactly Two

Every entity must have **exactly two** constructors.

### 1. Empty constructor (NHibernate mapping)

Required by the ORM. Always include the XML doc comment.

```csharp
/// <summary>
/// This constructor is used by NHibernate for mapping purposes.
/// </summary>
public User() { }
```

### 2. Parameterized constructor (entity creation)

Used by application code to create new instances.

```csharp
/// <summary>
/// Initializes a new instance with the specified values.
/// </summary>
public User(string email, string name)
{
    Email = email;
    Name = name;
}
```

**Rules for the parameterized constructor:**

- Include only the properties essential for creating a valid entity.
- **NEVER** include `Id` -- it is auto-generated by the base class.
- **NEVER** include `CreationDate` -- it is auto-assigned by the base class.
- **NEVER** include collections -- they are initialized at the property declaration.
- **NEVER** include navigation/reference properties unless required at creation time.

---

## GetValidator Override

Every entity must override `GetValidator()` to return its corresponding `FluentValidation` validator.

```csharp
public override IValidator GetValidator() => new UserValidator();
```

The validator class lives in the `validators/` directory within the same domain project. See the validators reference for details.

## Complete Entity Examples

### Simple entity (single property)

```csharp
namespace apsys.erp.domain.entities;

using apsys.core.domain;
using apsys.erp.domain.validators;
using FluentValidation;

/// <summary>
/// Represents a role that can be assigned to users.
/// </summary>
public class Role : AbstractDomainObject
{
    /// <summary>
    /// Gets or sets the name of the role.
    /// </summary>
    public virtual string Name { get; set; } = string.Empty;

    /// <summary>
    /// This constructor is used by NHibernate for mapping purposes.
    /// </summary>
    public Role() { }

    /// <summary>
    /// Initializes a new instance with the specified values.
    /// </summary>
    /// <param name="name">The name of the role.</param>
    public Role(string name)
    {
        Name = name;
    }

    public override IValidator GetValidator() => new RoleValidator();
}
```

### Medium entity (multiple properties, collection)

```csharp
namespace apsys.erp.domain.entities;

using apsys.core.domain;
using apsys.erp.domain.validators;
using FluentValidation;

/// <summary>
/// Represents a user in the system.
/// </summary>
public class User : AbstractDomainObject
{
    /// <summary>
    /// Gets or sets the email address.
    /// </summary>
    public virtual string Email { get; set; } = string.Empty;

    /// <summary>
    /// Gets or sets the display name.
    /// </summary>
    public virtual string Name { get; set; } = string.Empty;

    /// <summary>
    /// Gets or sets whether the user account is locked.
    /// </summary>
    public virtual bool Locked { get; set; }

    /// <summary>
    /// Gets or sets the roles assigned to this user.
    /// </summary>
    public virtual IList<Role> Roles { get; set; } = new List<Role>();

    /// <summary>
    /// This constructor is used by NHibernate for mapping purposes.
    /// </summary>
    public User() { }

    /// <summary>
    /// Initializes a new instance with the specified values.
    /// </summary>
    /// <param name="email">The email address.</param>
    /// <param name="name">The display name.</param>
    public User(string email, string name)
    {
        Email = email;
        Name = name;
    }

    public override IValidator GetValidator() => new UserValidator();
}
```

### Complex entity (many properties, dates)

```csharp
namespace apsys.erp.domain.entities;

using apsys.core.domain;
using apsys.erp.domain.validators;
using FluentValidation;

/// <summary>
/// Represents a prototype document with issue and expiration tracking.
/// </summary>
public class Prototype : AbstractDomainObject
{
    /// <summary>
    /// Gets or sets the prototype number.
    /// </summary>
    public virtual string Number { get; set; } = string.Empty;

    /// <summary>
    /// Gets or sets the date the prototype was issued.
    /// </summary>
    public virtual DateTime IssueDate { get; set; }

    /// <summary>
    /// Gets or sets the expiration date.
    /// </summary>
    public virtual DateTime ExpirationDate { get; set; }

    /// <summary>
    /// Gets or sets the current status.
    /// </summary>
    public virtual string Status { get; set; } = string.Empty;

    /// <summary>
    /// This constructor is used by NHibernate for mapping purposes.
    /// </summary>
    public Prototype() { }

    /// <summary>
    /// Initializes a new instance with the specified values.
    /// </summary>
    /// <param name="number">The prototype number.</param>
    /// <param name="issueDate">The date the prototype was issued.</param>
    /// <param name="expirationDate">The expiration date.</param>
    /// <param name="status">The current status.</param>
    public Prototype(string number, DateTime issueDate, DateTime expirationDate, string status)
    {
        Number = number;
        IssueDate = issueDate;
        ExpirationDate = expirationDate;
        Status = status;
    }

    public override IValidator GetValidator() => new PrototypeValidator();
}
```

## Anti-Patterns -- NEVER Do These

| Anti-Pattern                          | Why It Is Wrong                                             |
|---------------------------------------|-------------------------------------------------------------|
| ORM attributes (`[Table]`, `[Column]`) | Mapping is done via NHibernate `.hbm.xml` files, not annotations. |
| Serialization attributes (`[JsonProperty]`) | Serialization is handled at the API layer, not the domain.  |
| Persistence logic (`SaveToDatabase()`) | Persistence belongs in repositories, not entities.          |
| External dependencies / DI in entities | Entities are pure domain objects with no infrastructure concerns. |
| Non-virtual properties                | Breaks NHibernate proxy generation for lazy loading.        |
| Factory methods on the entity         | Use Domain Services for complex creation logic.             |
| Validation logic in property setters  | Use FluentValidation validators via `GetValidator()`.       |

---

## Generation Checklist

Use this checklist to verify every generated entity:

- [ ] Inherits `AbstractDomainObject`
- [ ] Namespace follows `{project}.domain.entities`
- [ ] All properties are `virtual` with correct defaults
- [ ] Empty constructor present with NHibernate XML doc comment
- [ ] Parameterized constructor present (no `Id`, no `CreationDate`, no collections)
- [ ] `GetValidator()` overridden, returning the matching validator
- [ ] Corresponding validator class created in `validators/`
- [ ] No ORM attributes, no serialization attributes, no persistence logic
- [ ] XML documentation on all public properties and constructors
